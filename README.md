# AuraGraph — AI Study Buddy
**Team Wowffulls | IIT Roorkee | Microsoft Azure AI Study Buddy Challenge**

AuraGraph converts your professor's slides and textbook PDFs into personalised, proficiency-aware study notes.  
It tracks which concepts you've mastered, lets you ask doubts that rewrite your notes in real-time (Note Mutation), and generates targeted exam practice pinpointing your weakest concepts.

---

## Azure Services Architecture

| Service | Role | Fallback |
|---|---|---|
| **Azure OpenAI GPT-4o** | Note generation, mutation, doubt answering, exam generation | Groq llama-3.3-70b |
| **Azure OpenAI Embeddings** (`text-embedding-3-large`) | Textbook chunk embeddings for semantic RAG | Local TF-IDF vectors |
| **Azure AI Vision** | Slide image OCR, figure captioning, diagram understanding | Groq vision (llama-4-scout) |
| **Azure AI Content Safety** | Screens all LLM output before showing to student | Pass-through (fail-open) |
| **Azure AI Search** | Vector RAG — stores & retrieves textbook+slide embeddings | numpy cosine similarity |
| **Azure Cosmos DB** | Persistent Cognitive Knowledge Graph (concept nodes + mastery) | SQLite |
| **Microsoft Semantic Kernel** | Orchestrates GPT-4o function calls for fusion, mutation, review | — |

> **Every Azure service has a local fallback.** The app runs fully offline with just `GROQ_API_KEY`.  
> Configure Azure services progressively — each one improves quality independently.

## Core Features

### 1. Semantic Fusion (Note Generation)
Upload slides (PDF/images) + optional textbook PDF → AuraGraph runs an 8-step pipeline:
1. Extract text (Azure AI Vision OCR for images, pdfplumber for PDFs)
2. Chunk + store in knowledge store
3. Embed textbook chunks (Azure OpenAI Embeddings / TF-IDF)
4. Analyse slide structure with GPT-4o → structured topic list
5. Retrieve relevant textbook context per topic (Azure AI Search / numpy)
6. Generate per-topic notes with GPT-4o (proficiency: Foundations / Practitioner / Expert)
7. Self-review accuracy pass
8. Persist notes + concept graph

### 2. Note Mutation ⭐
Student types a doubt → GPT-4o rewrites the exact note page to resolve it, backpropagating understanding into the notes. Powered by Microsoft Semantic Kernel.

### 3. Cognitive Knowledge Graph
Auto-extracted concept nodes with mastery status (struggling / partial / mastered).  
Persisted in **Azure Cosmos DB** or SQLite. Visualised as an animated galaxy graph in the UI.

### 4. Sniper Examiner
Targets questions at your weakest concepts from the knowledge graph. Generated by GPT-4o using your own slides as source material.

## Quick Start (local dev)

### Prerequisites
- Python 3.11+, Node 20+
- At minimum: `GROQ_API_KEY` (for LLM fallback) or `AZURE_OPENAI_*` keys

### Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env            # fill in your API keys
uvicorn main:app --reload --port 8000
```

### Frontend
```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

Open http://localhost:5173

## Security Notes

- `*.db` files are gitignored and must never be committed (contain password hashes).
- `bcrypt` is required. The app refuses to register users without it.
- `DEMO_ENABLED` defaults to `false`. Never enable in production.
- Session tokens are UUIDs with 7-day TTL. Invalidate by deleting rows from the `sessions` table.

## Deploy to Railway

1. Push to GitHub.
2. Create a Railway project → "Deploy from GitHub repo".
3. Set all env vars from `backend/.env.example` in the Railway dashboard.
4. Deploy the frontend separately (Vercel / Netlify) and set `CORS_ORIGINS` to your frontend URL.
